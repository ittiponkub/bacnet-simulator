BACnet Simulator — Help

Program guide and BACnet protocol reference.

Getting Started

Running the Program

python -m backend.main              # Desktop window
python -m backend.main --no-gui     # Browser at http://127.0.0.1:8736

First launch auto-creates config/devices.json with 50 default BMS points.

Start / Stop

Click Start (green, top-right) to activate BACnet engine. Device broadcasts I-Am and responds to read/write. Click Stop (red) to deactivate.

Status Bar

  • Start/Stop — green=running, red=stopped
  • Device name + instance
  • Connection status — Online/Offline

Sidebar

  • Add point — create new object
  • Export/Import CSV
  • Shutdown — close application

  • Units — °C, %, kW, etc.

BACnet Overview

BACnet (Building Automation and Control Networks) — ISO 16484-5 / ASHRAE 135. Protocol for HVAC, lighting, fire, access control, energy management.

Core Model

  • Device — one device on network, unique instance
  • Objects — data points (sensors, commands)
  • Properties — attributes (presentValue, objectName, units)
  • Services — read, write, discover

Commandable — Detail

Why WRITE_ACCESS_DENIED?

Writing presentValue of commandable object without priority is rejected. Must specify priority slot.

Write Flow

  1. Client writes value + priority (e.g. 8)
  2. Device stores in priority slot 8
  3. PV recalculated from highest non-NULL
  4. All NULL → PV = relinquish default

Example

All NULL, default=0      → PV=0
Write 75 at P8           → PV=75
Write 50 at P16          → PV=75 (P8 wins)
Relinquish P8 (NULL)     → PV=50 (P16)
Relinquish P16           → PV=0 (default)

ภาพรวม BACnet

BACnet (Building Automation and Control Networks) — ISO 16484-5 / ASHRAE 135 โปรโตคอลสำหรับ HVAC ไฟฟ้า ดับเพลิง access control จัดการพลังงาน

โมเดลหลัก

  • Device — อุปกรณ์บนเครือข่าย มี instance เฉพาะ
  • Objects — จุดข้อมูล (เซนเซอร์ คำสั่ง)
  • Properties — คุณสมบัติ (presentValue, objectName, units)
  • Services — อ่าน เขียน ค้นหา

Commandable — รายละเอียด

ทำไมได้ WRITE_ACCESS_DENIED?

เขียน presentValue ของ commandable object โดยไม่ระบุ priority จะถูกปฏิเสธ ต้องระบุ priority slot

ขั้นตอนเขียน

  1. Client เขียนค่า + priority (เช่น 8)
  2. อุปกรณ์เก็บใน priority slot 8
  3. คำนวณ PV จากช่องสำคัญสุดที่ไม่ NULL
  4. ทุกช่อง NULL → PV = relinquish default

ตัวอย่าง

ทุกช่องว่าง default=0       → PV=0
เขียน 75 ที่ P8             → PV=75
เขียน 50 ที่ P16            → PV=75 (P8 ชนะ)
Relinquish P8 (NULL)        → PV=50 (P16)
Relinquish P16              → PV=0 (default)
